# Design.md

## 目标

本文件用于约束刘海屏工具项目的视觉规范与交互表现，作为后续实现 UI、组件和动效时的统一参照。



## 整体设计方向

- 产品整体是 `深色半透明工具箱` 风格。
- 主体面板强调低干扰、悬浮、贴顶、轻量控制感，而不是高彩度卡片化。
- 视觉重心是：
  - 深色容器
  - 白色文字与图标
  - 白色透明叠加层级
  - 小圆角、高密度信息编排
- 彩色主要来自内容本身，例如音乐 App Logo、文件缩略图、封面图，不来自基础控件底色。



## 颜色规范

### 文字颜色

- 文字主色：`#FFFFFF`
  
  - 用于标题、主标签、主按钮文字、导航文字、Tab 文字、设置项名称

- 文字次要色：`rgba(255, 255, 255, 0.7)`
  
  - 等价：`#FFFFFFB3`
  - 用于副标题、说明文字、歌手名、累计说明、弱化按钮文字、模型名称

- 文字辅助色：`rgba(255, 255, 255, 0.5)`
  
  - 等价：`#FFFFFF80`
  - 用于文件类型、次级元信息、引导型弱提示

- 文字占位色 / 空状态色：`rgba(255, 255, 255, 0.3)`
  
  - 等价：`#FFFFFF4D`
  - 用于输入框 placeholder、空状态提示

- 危险 / 停止态文字色：`rgba(255, 75, 75, 0.7)`
  
  - 用于 `停止专注`、`停止休息` 等带终止语义的操作

### 基础表面颜色

- 主输入框底色：`#1A1A1A`
  
  - 用于 AI 输入框内层主承载容器

- 下拉浮层底色：`#1F1F1F`
  
  - 用于模型切换弹层、悬浮选择层

- 选中 segment / 当前项底色：`#000000`
  
  - 用于顶部分段控件中的当前激活项

- 导航选中底色：`rgba(255, 255, 255, 0.08)`
  
  - 用于设置面板左侧当前导航项

- 次级容器底色：`rgba(255, 255, 255, 0.10)`
  
  - 用于顶部 `Tab` 外层承载容器、番茄钟时长切换容器

- 轻量内嵌选择器 / 选中条目底色：`rgba(255, 255, 255, 0.08)`
  
  - 用于输入框内部模型切换胶囊
  - 用于下拉列表中的选中项底色
  - 不应误写为“所有输入 / 下拉主承载底色”

- 未选中勾选框底色：`rgba(255, 255, 255, 0.20)`

- 分割线颜色：`rgba(255, 255, 255, 0.10)`

### 描边颜色

- 浅描边主规范：`rgba(255, 255, 255, 0.20)`
  
  - 用于输入框、设置弹层、部分环形描边

- 弱描边规范：`rgba(255, 255, 255, 0.10)`
  
  - 用于内容容器边界、较弱结构分隔



## 字体规范

### 中文字体

- 首选：`PingFang SC`
- 常见字重：
  - `Regular`
  - `Medium`

适用场景：

- 绝大多数中文标题、标签、设置项、说明文字

### 英文 / 数字字体

- 首选：`SF Pro`
- 常见字重：
  - `Regular`
  - `Medium`

适用场景：

- 时间数字
- 音乐进度时间
- 英文模型名
- 英文快捷键文案

### 字号层级

- `24px`
  
  - 大号核心数字
  - 用于番茄钟主计时

- `13px`
  
  - 主操作层级
  - 用于导航项、设置项、按钮文字、歌曲名、常规主标签

- `12px`
  
  - 次级信息层级
  - 用于歌手名、累计说明、时长选项、空状态弱提示中的部分场景

- `11px`
  
  - 极密信息层级
  - 用于文件列表的文件名 / 类型信息

- `10px`
  
  - 辅助时间信息
  - 用于音乐进度条两端时间

### 行高与排版倾向

- 大部分文本使用自然行高或紧凑单行布局。
- 本项目以单行信息压缩为主，常见搭配是：
  - 单行标题
  - 单行副信息
  - 单行按钮
- 对超长文字采用截断，不做大面积自动换行。

## 结构与布局规范

### 顶部主面板

- 主面板宽度常见为：
  
  - `580px` 横向展开面板
  - `320px` 竖向/紧凑态面板

- 顶部安全区占位：
  
  - 常见 `safeAreaInsets` 宽度：`186px`
  - 作为顶部刘海避让的中心留白区

- 顶部结构一般分为三段：
  
  - 左：功能切换
  - 中：安全区 / 主内容中心
  - 右：设置入口

### Tab 切换结构

- 外层容器：
  
  - 高度 `31px`
  - 圆角 `8px`
  - 背景 `rgba(255,255,255,0.10)`

- 内层项：
  
  - 高度 `27px`
  - 左右间距紧凑
  - 当前激活项使用 `#000000`

### 列表与文件卡片

- 文件列表使用横向压缩布局。

- 单个文件项由三部分构成：
  
  - 缩略图
  - 文件名
  - 类型信息

- 文件项内部信息结构：
  
  - 文件名：主色白
  - 类型：白色 `50%`

## 交互状态规范

### 按钮 / 热区状态层

每个可操作按钮都必须拥有独立热区，不允许仅以图标本身作为命中范围。

- `Hover` 状态层：
  
  - `rgba(255, 255, 255, 0.08)`

- `Active / Pressed` 状态层：
  
  - `rgba(255, 255, 255, 0.06)`

规则：

- 使用“白色透明叠加层”表达交互反馈。
- 不直接改写组件的结构底色。
- 已选中项在 `Hover` / `Active` 时，仍保留选中底色，再叠加对应状态层。

### 选中态

- Tab / Segment 当前项：
  
  - 使用 `#000000` 作为选中底色

- 设置导航当前项：
  
  - 使用 `rgba(255,255,255,0.08)` 作为选中底色

- 下拉列表当前项：
  
  - 使用 `rgba(255,255,255,0.08)` 作为选中底色

### Checkbox

- 未选中：
  
  - `rgba(255,255,255,0.20)` 底色
  - 小圆角 `4px`

- 已选中：
  
  - 采用深色底配白色勾选图形
  - 保持高对比，不做灰弱化

## 滚动条规范

当内容超出展示区域时，必须提供自定义滚动条。

- 宽度：`3pt`
- 颜色：`rgba(255, 255, 255, 0.25)`
- 默认状态：隐藏
- 鼠标进入可交互滚动区域时：显示
- 鼠标移出可交互区域后：消失

规则：

- 滚动条属于临时辅助反馈，不可常驻抢占视觉注意力。
- 滚动条样式必须轻、细、弱化，不能破坏整体深色悬浮感。

  - 外层 `rgba(255,255,255,0.10)`

### 音乐模块

  - `13px`

  - `10px`

- 进度条轨道：
  
  - `rgba(255,255,255,0.20)`

  - `#FFFFFF`

## 实现纪律

- 不要擅自把深色面板改成纯平浅色系统。
- 不要引入额外强调色去替换当前白色透明体系。
- 不要将局部组件色值误提升为全局 token。
- 当某个容器实际使用图像 / 矢量 BG 资产时，优先遵循视觉还原，不强行替换成单一纯色。
- 后续若新增设计规范，应优先延续：
  - 深色容器
  - 白色文字
  - 白色透明状态层
  - 小圆角、多层级弱分隔
