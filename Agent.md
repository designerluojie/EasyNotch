# AGENTS.md

## 目标

构建一个 macOS 刘海屏工具箱应用，把顶部刘海区或顶部锚点区做成低打扰、低能耗、可常驻的效率入口。
默认行为：先执行，保持改动聚焦，优先保证顶部锚点适配、多屏稳定性和关闭态低资源占用。

## 产品范围

- 平台：仅支持 macOS。
- 兼容目标：最低兼容 `macOS 13+`，推荐以 `macOS 14+` 作为主要体验优化目标。
- 形态目标：同时支持有刘海内置屏与无刘海设备/外接屏的顶部入口形态。
- 顶部策略：有刘海设备使用 notch shell；无刘海设备退化为顶部中心 handler 或模拟刘海入口。
- 首次启动：展示一次新手引导，默认只出现一次。
- 主模块：
  - 音乐控制（默认模块）
  - 文件暂存
  - AI Chat
  - 剪贴板
  - 番茄钟
- 配套模块：
  - 设置
- AI Chat 支持用户自配 `DeepSeek`、`Qwen`、`ChatGPT`、`Gemini`。
- AI Chat 输入支持文字与图片；如果模型不支持图片，需要明确提示不支持。
- 音乐控制目标支持启动或接入：
  - Apple Music
  - 网易云音乐
  - QQ 音乐
  - 酷狗音乐
  - 汽水音乐
  - Spotify
- 剪贴板首期支持纯文字、富文本、图片、标准SVG、Figma图形、Figma文字、文件/多文件/文件夹。
- 文件暂存支持文件拖入保存与拖出使用，并支持删除。
- 番茄钟支持 `25 / 45 / 60` 分钟专注时长，并显示今日累计专注时长。
- 设置页需覆盖功能排序、AI 配置、剪贴板保存数量、自动清理、文件暂存自动清理、启动项、全局快捷键、模拟刘海、动效模式与动效速度。

## 产品优先级

1. 保持主链路顺畅：唤起 → 查看/操作模块 → 收起。
2. 优先保证顶部锚点适配、多屏迁移与全屏场景稳定。
3. 优先采用低能耗、关闭即休眠的行为。
4. 不写死任何具体机型的刘海尺寸或几何参数。
5. 模块在不可见时不保留无意义刷新、预热或后台常驻连接。

## 关键产品规则

- 同一时间只允许一个完整 active panel 实例处于展开态。
- 多屏场景下，每块屏幕可以保留轻量唤起入口，但完整面板必须单实例迁移。
- 从 A 屏切换到 B 屏时，A 屏先收起，再将同一面板实例迁移到 B 屏。
- 外接屏、合盖、分辨率变化、菜单栏自动隐藏、全屏切换后，面板位置和可见策略必须正确更新。
- 关闭态必须接近无感知资源占用。
- AI Chat 不提供默认模型、不提供默认 API 服务、不做后台预热。
- 对话历史可保留，但不能让历史管理反向拉高关闭态资源成本。
- 音乐模块允许将不同播放器接入拆成统一控制层，但不能把未验证能力写成已稳定完成。
- 所有的面板展开、收起、尺寸变化，都需要有动效，动效参数使用统一的参数来维护，并且支持用户在设置里配置动效参数选项。

## 输入与行为约束

- 音乐控制在未播放时应引导启动支持的播放器。
- 音乐播放中：
  - 未展开时展示播放源 Logo 与动态播放提示
  - 展开后展示封面、歌曲名、歌手、进度，并支持播放控制
- 文件暂存支持拖入单个或多个文件。
- 文件暂存默认采用引用式暂存；第一版不做文件副本仓库。
- AI Chat 未配置 API 时必须引导配置，而不是显示空白主界面。
- 剪贴板历史展示来源图标、时间和内容摘要。
- 剪贴板保存上限由设置控制，候选值为 `5 / 10 / 15 / 20 / 30 / 50`。
- 番茄钟支持暂停专注与停止专注。
- 全局快捷键默认值为 `Command + Option + T`。
- 模拟 Mac 刘海屏的选项仅对非刘海屏设备生效。

## 技术约束

- 主技术栈：Swift / SwiftUI / AppKit。
- 窗口层：`NSPanel` + `borderless` + `nonactivatingPanel`。
- 屏幕几何：基于 `NSScreen.safeAreaInsets.top`、`auxiliaryTopLeftArea`、`auxiliaryTopRightArea`。
- 顶部适配：围绕“顶部锚点区”实现，不围绕固定刘海轮廓写死布局。
- 多屏协调：需要独立的显示拓扑、屏幕画像、锚点计算与面板协调层。
- 音乐信息与控制：优先复用 `MediaRemote / Now Playing` 相关链路，并允许用本地时间轴矫正第三方播放器进度漂移。
- 文件暂存：优先采用 `bookmarkData` 保存原文件引用。
- 剪贴板：基于 `NSPasteboard` 监听与本地持久化；如使用轮询，必须配合节流、休眠和自写回避策略。
- AI 提供商配置与敏感信息必须安全持久化，不能明文散落在普通配置存储中。

## 低能耗纪律

- 事件驱动优先，不做无必要的持续高频监听。
- 面板关闭后不刷新 AI、音乐详情、番茄钟 UI、剪贴板列表和其他模块内容。
- 动画只在用户主动触发的状态切换中运行，不做待机呼吸、常驻流光或持续毛玻璃重绘。
- 多屏数量增加时，不能让完整面板实例线性扩张。
- 非当前激活屏幕仅允许保留轻量入口，不允许同时展开多个完整功能容器。
- 对于必须采用轮询的链路，必须明确暂停条件、节流策略和唤醒恢复策略。

## 明确的非目标

- 本期不做跨平台支持。
- 不做围绕固定设备刘海宽高的硬编码适配。
- 不做同时展开多个完整面板实例。
- 不做后台 AI 预热、默认模型托管或隐式联网保活。
- 不做与当前功能清单无关的额外模块扩张。
- 不把旧文档中已经滞后的截图、翻译、提醒事项/日历摘要直接当作当前范围内模块。
- 不为了视觉效果引入明显持续 CPU / GPU 占用。
